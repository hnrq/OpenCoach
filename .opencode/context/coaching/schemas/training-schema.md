# Training File Schema

## Exercise

_Object containing the following properties:_

| Property        | Description                                                                  | Type                                       |
| :-------------- | :--------------------------------------------------------------------------- | :----------------------------------------- |
| **`name`** (\*) | Exercise name exactly as returned by ExerciseDB API                          | `string`                                   |
| `warmup_sets`   | Progressive warmup sets before working sets                                  | _Array of_ [WarmupSet](#warmupset) _items_ |
| `working_sets`  | Legacy field — use sets instead                                              | `number`                                   |
| `sets`          | Number of working sets                                                       | `number`                                   |
| `reps`          | Rep target. Use string for ranges ('6-8'), number for fixed (8)              | `string \| number`                         |
| `rpe`           | RPE target. Use string for ranges ('7–8'). Main lifts: 7–8, Accessories: 7–8 | `string \| number`                         |
| `note`          | Coaching cue for the athlete, e.g. 'Drive knees out at bottom'               | `string`                                   |
| `bodyPart`      | From ExerciseDB: body part category, e.g. 'chest', 'legs'                    | `string`                                   |
| `equipment`     | From ExerciseDB: equipment required, e.g. 'barbell', 'dumbbell'              | `string`                                   |
| `target`        | From ExerciseDB: primary target muscle, e.g. 'pectorals', 'quads'            | `string`                                   |

_(\*) Required._

## MetabolicPrimer

_Object containing the following properties:_

| Property        | Description                                                   | Type     |
| :-------------- | :------------------------------------------------------------ | :------- |
| **`name`** (\*) | Circuit name, e.g. 'AMRAP Circuit' or 'Tabata Protocol'       | `string` |
| `format`        | Format type: AMRAP \| Tabata \| EMOM \| Interval              | `string` |
| `duration_min`  | Total primer duration in minutes, typically 6–10              | `number` |
| `protocol`      | Work/rest prescription, e.g. '20s work / 10s rest × 8 rounds' | `string` |
| `instructions`  | Detailed instructions for the athlete                         | `string` |
| `rpe_target`    | Target RPE for the primer, usually 8–9                        | `number` |

_(\*) Required._

## Session

_Object containing the following properties:_

| Property                 | Description                                                                                          | Type                                                                                                                                                                                                                                                                                                                                                                            |
| :----------------------- | :--------------------------------------------------------------------------------------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `id`                     | Optional identifier for the session                                                                  | `string`                                                                                                                                                                                                                                                                                                                                                                        |
| **`day`** (\*)           | Weekday for this session, e.g. 'monday', 'wednesday'                                                 | `string`                                                                                                                                                                                                                                                                                                                                                                        |
| `focus`                  | Session label, e.g. 'Upper Body Push', 'Legs', 'Full Body'                                           | `string`                                                                                                                                                                                                                                                                                                                                                                        |
| `scheduling_note`        | Note about scheduling, e.g. 'Must be 48h after sport'                                                | `string`                                                                                                                                                                                                                                                                                                                                                                        |
| `estimated_duration_min` | Estimated session duration in minutes                                                                | `number`                                                                                                                                                                                                                                                                                                                                                                        |
| `metabolic_primer`       | REQUIRED by Michaels methodology. Every session must open with a metabolic primer (AMRAP or Tabata). | [MetabolicPrimer](#metabolicprimer)                                                                                                                                                                                                                                                                                                                                             |
| `exercises`              | Working exercises from ExerciseDB only. Order: compound lifts first, accessories after.              | _Array of_ [Exercise](#exercise) _items_                                                                                                                                                                                                                                                                                                                                        |
| `alternating_weeks`      | Use for A/B week alternating programmes. Keys: 'weekA', 'weekB'                                      | _Object with dynamic keys of type_ `string` _and values of type_ _Object with properties:_<ul><li><code>label</code>: <code>string</code> - Week label, e.g. 'Week A' or 'Week B'</li><li><code>metabolic_primer</code>: <a href="#metabolicprimer">MetabolicPrimer</a></li><li><code>exercises</code>: <i>Array of</i> <a href="#exercise">Exercise</a> <i>items</i></li></ul> |

_(\*) Required._

## Training

_Object containing the following properties:_

| Property            | Description                                                                                      | Type                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| :------------------ | :----------------------------------------------------------------------------------------------- | :-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `athlete`           | Athlete name — copy from profile.json                                                            | `string`                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| `date`              | Session date in YYYY-MM-DD format                                                                | `string` (_regex: `/^\d{4}-\d{2}-\d{2}$/`_)                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| `methodology`       | Always 'Michaels'                                                                                | `string`                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| `sport_context`     | Passed in by Head Coach — do not derive independently                                            | _Object with properties:_<ul><li><b><code>sport</code></b> (\*): <code>string</code> - Athlete's primary sport, e.g. 'Futsal', 'Muay Thai', 'Bodybuilding'</li><li><code>mapped_to</code>: <code>string</code> - Nearest sport from the Sport Goal Reference table if the athlete's sport is unlisted</li><li><code>training_focus</code>: <code>string</code> - Primary physical qualities to develop, e.g. 'explosive power + agility'</li><li><code>movement_patterns</code>: <code>string</code> - Dominant movements in the sport, e.g. 'lateral cuts, sprinting, kicking'</li><li><code>injury_areas</code>: <code>string \| Array<string></code> - Body areas to monitor and avoid overloading, e.g. ['knees', 'ankles']</li><li><code>energy_system</code>: <code>string</code> - Aerobic/anaerobic profile, e.g. '70% aerobic / 30% anaerobic'</li><li><code>nutrition_strategy</code>: <code>string</code> - Sport-specific dietary guidance, e.g. 'carb-dominant; match-day carb loading'</li><li><code>notes</code>: <code>string</code> - Additional sport-specific coaching notes</li></ul> |
| `constraints`       | From profile.json: no_jumping, gym_only, etc. Copy verbatim from profile.                        | `Record<string, unknown>`                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| `progression`       | RPE-based progression rules: rpe_targets (main_lifts, accessories) and advancement rule string   | `Record<string, unknown>`                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| `weekly_schedule`   | Map of weekday to session focus label, e.g. { monday: 'Upper Body Push', wednesday: 'Legs' }     | `Record<string, string>`                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| `warmup_template`   | General warmup structure: optional cardio, mobility drills, ramp sets for first lift             | `Record<string, unknown>`                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| `sessions`          | Array of training sessions (preferred format). Must include a metabolic_primer in every session. | _Array of_ [Session](#session) _items_                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| `circuits`          | Legacy circuit format — only use when migrating old data. New plans must use sessions[].         | _Array of objects:_<ul><li><b><code>name</code></b> (\*): <code>string</code> - Circuit name — legacy format, use sessions[] instead</li><li><b><code>exercises</code></b> (\*): <i>Array of objects:</i><ul><li><b><code>name</code></b> (\*): <code>string</code></li><li><b><code>sets</code></b> (\*): <code>number</code></li><li><b><code>reps</code></b> (\*): <code>string</code></li><li><code>rpe</code>: <code>number</code></li></ul></li></ul>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| `progression_rule`  | Legacy field — put progression rules in progression.rule instead                                 | `string`                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| `progression_logic` | Legacy field — put progression rules in progression.rule instead                                 | `string`                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |

_All properties are optional._

## WarmupSet

_Object containing the following properties:_

| Property        | Description                                | Type     |
| :-------------- | :----------------------------------------- | :------- |
| **`reps`** (\*) | Rep count for this warmup set              | `number` |
| **`load`** (\*) | Load description, e.g. '40kg' or '50% 1RM' | `string` |
| `note`          | Optional coaching note for this warmup set | `string` |

_(\*) Required._
