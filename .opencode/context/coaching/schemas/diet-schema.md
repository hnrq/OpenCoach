# Diet File Schema

## DayType

_Object containing the following properties:_

| Property         | Description                                                                                                          | Type                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                     |
| :--------------- | :------------------------------------------------------------------------------------------------------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `label`          | Human-readable label for this day type, e.g. 'Gym + Futsal Day'                                                      | `string`                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| `note`           | Coaching note about this day type                                                                                    | `string`                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| `timing_note`    | When meals should be eaten relative to training                                                                      | `string`                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| `days`           | Weekdays this type applies to, e.g. ['monday', 'wednesday', 'friday']                                                | `Array<string>`                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| `total_targets`  |                                                                                                                      | _Object with properties:_<ul><li><b><code>calories</code></b> (\*): <code>number</code> - Total kilocalories for this day type</li><li><b><code>protein_g</code></b> (\*): <code>number</code> - Always = macro_rules.protein_g (LBM kg × 2.2). Constant across all day types.</li><li><b><code>carbs_g</code></b> (\*): <code>number</code> - Variable carb target. Training days: 150–250g typical. Rest days: cut 20–40% from training day.</li><li><b><code>fats_g</code></b> (\*): <code>number</code> - Always ≥ macro_rules.fat_g (bodyweight kg × 0.8). Constant across all day types.</li></ul> |
| `macros_g`       |                                                                                                                      | _Object with properties:_<ul><li><b><code>protein</code></b> (\*): <code>number</code> - Grams of protein. Always = macro_rules.protein_g (LBM kg × 2.2). Constant across all day types.</li><li><b><code>carbs</code></b> (\*): <code>number</code> - Variable carb target. Training days: 150–250g typical. Rest days: cut 20–40% from training day baseline.</li><li><b><code>fat</code></b> (\*): <code>number</code> - Grams of fat. Always ≥ macro_rules.fat_g (bodyweight kg × 0.8). Constant across all day types.</li></ul>                                                                     |
| `meals`          | Full meal breakdown — populate when specifying per-meal foods and macros                                             | _Array of_ [Meal](#meal) _items_                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| `meal_templates` | Meal slot names in order when not specifying full meals, e.g. ['Breakfast', 'Pre-workout', 'Post-workout', 'Dinner'] | `Array<string>`                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |

_All properties are optional._

## Diet

_Object containing the following properties:_

| Property          | Description                                                                                                                                | Type                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| :---------------- | :----------------------------------------------------------------------------------------------------------------------------------------- | :-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `athlete`         | Athlete name — copy from profile.json                                                                                                      | `string`                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| `date`            | Session date in YYYY-MM-DD format                                                                                                          | `string` (_regex: `/^\d{4}-\d{2}-\d{2}$/`_)                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| `methodology`     | Always 'Michaels'                                                                                                                          | `string`                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| `sport_context`   | Passed in by Head Coach — do not derive independently                                                                                      | _Object with properties:_<ul><li><b><code>sport</code></b> (\*): <code>string</code> - Athlete's primary sport, e.g. 'Futsal', 'Muay Thai', 'Bodybuilding'</li><li><code>mapped_to</code>: <code>string</code> - Nearest sport from the Sport Goal Reference table if the athlete's sport is unlisted</li><li><code>training_focus</code>: <code>string</code> - Primary physical qualities to develop, e.g. 'explosive power + agility'</li><li><code>movement_patterns</code>: <code>string</code> - Dominant movements in the sport, e.g. 'lateral cuts, sprinting, kicking'</li><li><code>injury_areas</code>: <code>string \| Array<string></code> - Body areas to monitor and avoid overloading, e.g. ['knees', 'ankles']</li><li><code>energy_system</code>: <code>string</code> - Aerobic/anaerobic profile, e.g. '70% aerobic / 30% anaerobic'</li><li><code>nutrition_strategy</code>: <code>string</code> - Sport-specific dietary guidance, e.g. 'carb-dominant; match-day carb loading'</li><li><code>notes</code>: <code>string</code> - Additional sport-specific coaching notes</li></ul> |
| `targets`         | Legacy field — ignore for new plans                                                                                                        | `Record<string, unknown>`                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| `inputs`          | Derived context from Head Coach: michaels_floors, sport_context passed as structured data                                                  | `Record<string, unknown>`                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| `macro_rules`     | Michaels floors: protein_g_per_kg_lbm: 2.2, fat_g_per_kg_bodyweight: 0.8, protein_g: <LBM×2.2>, fat_g: <weight×0.8>                        | `Record<string, unknown>`                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| `day_types`       | Carb-cycling day types (preferred over daily_targets). Common keys: gymOnly, legsDay, futsalGym, restDay                                   | _Object with dynamic keys of type_ `string` _and values of type_ [DayType](#daytype)                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| `daily_targets`   | Legacy flat targets — only use when migrating old data. New plans must use day_types.                                                      | _Object with properties:_<ul><li><b><code>calories</code></b> (\*): <code>number</code></li><li><b><code>protein_g</code></b> (\*): <code>number</code></li></ul>                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| `weekly_schedule` | Map of weekday to day type key, e.g. { monday: 'gymOnly', tuesday: 'futsalGym', sunday: 'restDay' }                                        | `Record<string, string>`                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| `michaels_floors` | Computed macro floors passed in by Head Coach: protein_floor_g (LBM×2.2), fat_floor_g (weight×0.8)                                         | `Record<string, unknown>`                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| `adjustments`     | Caloric adjustments with rationale, e.g. { reason: 'WROC below -0.3', carbs_delta_g: +20 }                                                 | `Record<string, unknown>`                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                 |
| `rationale`       | One sentence per reason the plan differs from the prior session. E.g. 'Carbs raised 20g — WROC below −0.3 kg/wk indicating under-recovery' | `Array<string>`                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |

_All properties are optional._

## Meal

_Object containing the following properties:_

| Property         | Description                                                          | Type                      |
| :--------------- | :------------------------------------------------------------------- | :------------------------ |
| **`meal`** (\*)  | Meal name, e.g. 'Breakfast', 'Pre-workout', 'Post-workout', 'Dinner' | `string`                  |
| `timing`         | When to eat relative to training, e.g. '60 min pre-workout'          | `string`                  |
| **`foods`** (\*) | List of foods from profile.json food_preferences.primary only        | `Array<string>`           |
| `macros`         | Macro breakdown for this meal — omit if using meal_templates instead | [MealMacros](#mealmacros) |

_(\*) Required._

## MealMacros

_Object containing the following properties:_

| Property             | Description                      | Type     |
| :------------------- | :------------------------------- | :------- |
| **`calories`** (\*)  | Total kilocalories for this meal | `number` |
| **`protein_g`** (\*) | Grams of protein                 | `number` |
| **`carbs_g`** (\*)   | Grams of carbohydrates           | `number` |
| **`fats_g`** (\*)    | Grams of fat                     | `number` |

_(\*) Required._
